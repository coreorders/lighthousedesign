# 제품 요구사항 정의서 (PRD)

## [1] 프로젝트 개요
* 제품명: Light House Design 고객 소통 플랫폼 (가칭)
* 목적: 인테리어 공사 현장별 진행 상황(사진, 일정, 메모)을 고객에게 달력 형태로 투명하게 공유하여 신뢰도를 높이고 소통을 원활하게 함
* 주요 특징:
    * 깃헙 페이지스(GitHub Pages)를 통한 프론트엔드 배포 (서버 호스팅 비용 제로)
    * 개인 컴퓨터(로컬 PC)를 활용한 도커(Docker) 기반 백엔드 및 데이터 저장소 운영
    * 현장별 URL 분리 및 간이 비밀번호 접근 제어

## [2] 시스템 아키텍처 및 데이터 흐름
* 프론트엔드 (GitHub Pages):
    * 사용자가 접속하는 정적 웹사이트 인터페이스 (HTML, CSS, JavaScript)
    * 로컬 서버의 API 주소(터널링 주소)로 데이터를 요청하여 화면에 렌더링
* 백엔드 & DB (내 컴퓨터 - Docker 환경):
    * 컨테이너 1 (API 서버): 프론트엔드의 요청을 처리하고 DB 데이터 전달 및 사진 업로드 처리 (Node.js 또는 Python 등)
    * 컨테이너 2 (Database): 현장 정보, 일정, 사진 경로, 한줄 메모, 비밀번호 등을 저장 (PostgreSQL 또는 MySQL)
    * 컨테이너 3 (역방향 프록시 및 터널링): 외부 인터넷에서 내 컴퓨터로 안전하게 접속할 수 있도록 암호화(HTTPS) 경로 제공 (Cloudflare Tunnel 또는 Ngrok)

## [3] 핵심 기능 요구사항 (Functional Requirements)

### 1. 고객용 화면 (Client View)
* 현장별 고유 접근 링크:
    * `https://[깃헙계정].github.io/[현장명]` 또는 쿼리스트링 형태(`?site=현장명`)로 진입
* 비밀번호 확인 창:
    * 현장 페이지 접속 시 가장 먼저 비밀번호 입력 창 노출 (단일 글자 또는 문자열 간이 인증)
    * 인증 성공 시에만 내부 데이터 로드
* 달력 기반 대시보드:
    * 달력 내 날짜별 공사 일정 표시 (예: 타일 작업, 도배 등)
    * 사진이 등록된 날짜에는 달력 칸에 작은 썸네일 또는 아이콘 표시
    * 날짜나 사진 클릭 시 해당 날짜에 업로드된 사진들을 크게 볼 수 있는 모달 창(슬라이드 뷰어) 오픈
* 한줄 메모 게시판:
    * 달력 하단 영역에 고객과 관리자가 자유롭게 한줄평이나 요청사항을 남길 수 있는 실시간 방명록 형태의 공간 제공
    * 고객은 메모 작성만 가능하며, 작성 후 수정 및 삭제는 불가능
    * 메모 수정 및 삭제 권한은 관리자에게만 제공

### 2. 관리자 기능 (Admin Dashboard)
* 관리자 인증:
    * 웹사이트 내 [관리자 로그인] 버튼 또는 별도 경로를 통해 최고 관리자 인증
* 현장 관리 (프로젝트 생성):
    * 새로운 인테리어 현장 추가 및 해당 현장의 접근 비밀번호 설정/수정
    * 공사가 완료된 현장은 관리자가 상태를 "완료"로 변경 가능
    * 완료된 현장은 관리자가 데이터를 보관하여 고객 링크를 유지하거나, 데이터를 삭제하여 고객 접근을 종료할 수 있음
* 날짜별 공사 콘텐츠 관리:
    * 달력에서 원하는 날짜를 선택하여 당일 진행된 현장 사진 업로드 (드래그 앤 드롭 지원, 로컬 서버 내 폴더에 저장)
    * 사진 업로드 용량은 애플리케이션 차원에서 제한하지 않음
    * 특정 날짜의 공사 일정 등록, 수정 및 삭제
* 하단 고정 공지글 작성:
    * 달력 맨 하단 영역에 관리자가 전체 공사 현황 요약, 안내 사항 등을 작성하여 고정하는 기능

## [4] 데이터베이스(DB) 간이 구조 설계
* 현장 테이블 (Sites):
    * 현장 ID, 현장명, 접근 비밀번호, 하단 고정 공지 내용, 현장 상태(진행중/완료/삭제됨), 생성일, 완료일
* 일정 및 사진 테이블 (Calendar_Data):
    * 데이터 ID, 현장 ID, 해당 날짜, 일정 텍스트, 저장된 사진 파일 경로 목록
* 한줄 메모 테이블 (Memos):
    * 메모 ID, 현장 ID, 작성자 구분, 메모 내용, 작성 시간, 관리자 삭제 여부

## [5] 기술적 도전 과제 및 해결 방안
* 도커(Docker) 기반 안전성 확보:
    * 로컬 본체 시스템 환경과 분리된 독립 컨테이너를 사용하여 보안성 향상
    * `docker-compose`를 활용해 API 서버, DB, 터널링 프로그램을 원 클릭으로 동시 제어
* 외부 인터넷 통신 (터널링):
    * 로컬 PC의 공인 IP 노출 및 복잡한 포트포워딩 설정 없이 Cloudflare Tunnel 또는 Ngrok을 활용해 외부 접근을 안전하게 허용
* 저장공간 관리:
    * 사진 업로드 용량을 제한하지 않으므로 로컬 디스크 용량 모니터링 및 오래된 현장 데이터 정리 기능이 필요
    * 완료된 현장은 관리자 판단에 따라 보관 또는 삭제할 수 있도록 하여 장기 저장 부담을 줄임
